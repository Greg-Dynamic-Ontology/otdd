# Ontology Test-Driven Development (OTDD)

Ontology Test-Driven Development (OTDD) is an engineering methodology for
semantic engineering.

OTDD provides a disciplined approach for designing, implementing,
verifying, and evolving ontologies and related semantic artifacts.

## Current Status

This repository contains the evolving OTDD methodology.

The methodology is being developed through practical application on real
projects rather than by theoretical design alone. The first reference
implementation is the **UAD 3.6 Validation** project.

## Repository Organization

```text
ontologies/
    Defines the semantic vocabulary used by OTDD.

metadata/
    Contains RDF instance data describing the engineering state of an
    OTDD project.

examples/
    Contains example instances used to demonstrate and verify semantic
    competencies.

docs/
    Human-readable documentation and generated projections.
```

## Guiding Principles

- Derivation, not divination.
- Meaning before implementation.
- Traceability over ceremony.
- Engineering knowledge belongs in the ontology.
- Human-readable artifacts are projections of semantic engineering
  knowledge.

## Relationship to UAD

The UAD 3.6 Validation project serves as the first reference
implementation of OTDD.

Experience gained during the UAD project is distilled into the OTDD
methodology. Improvements to the methodology are incorporated here as
they are discovered.
